在C++中，函数的参数传递方式主要有两种：传值参数和传引用参数。它们的主要区别在于是否直接操作原始数据。以下是这两种参数传递方式的详细说明和示例：

1. 传值参数

  • 定义：将实参的值复制给形参，函数内部对形参的修改不会影响实参。
  • 特点：
    • 形参是实参的副本。
    • 函数内部对形参的修改不会影响实参。
    • 适用于不需要修改实参的场景。

示例

#include <iostream>
using namespace std;

// 传值参数
void increment(int x) {
    x++;  // 修改形参
    cout << "Inside function: " << x << endl;  // 输出：11
}

int main() {
    int a = 10;
    increment(a);  // 调用函数，传递 a 的值
    cout << "Outside function: " << a << endl;  // 输出：10（a 未被修改）
    return 0;
}

2. 传引用参数

  • 定义：将实参的引用传递给形参，函数内部对形参的修改会直接影响实参。
  • 特点：
    • 形参是实参的别名。
    • 函数内部对形参的修改会直接影响实参。
    • 适用于需要修改实参或避免复制的场景。

示例

#include <iostream>
using namespace std;

// 传引用参数
void increment(int& x) {
    x++;  // 修改形参（即修改实参）
    cout << "Inside function: " << x << endl;  // 输出：11
}

int main() {
    int a = 10;
    increment(a);  // 调用函数，传递 a 的引用
    cout << "Outside function: " << a << endl;  // 输出：11（a 被修改）
    return 0;
}

3. 传引用参数的优势

  • 避免复制：对于大型对象（如结构体、类对象），传引用可以避免复制的开销。
  • 修改实参：可以直接修改实参的值。

示例：避免复制

#include <iostream>
#include <vector>
using namespace std;

// 传引用参数，避免复制
void printVector(const vector<int>& vec) {
    for (int num : vec) {
        cout << num << " ";
    }
    cout << endl;
}

int main() {
    vector<int> numbers = {1, 2, 3, 4, 5};
    printVector(numbers);  // 传递 vector 的引用，避免复制
    return 0;
}

4. 综合示例

以下是一个完整的C++程序，演示了传值参数和传引用参数的使用：

#include <iostream>
using namespace std;

// 传值参数
void incrementByValue(int x) {
    x++;
    cout << "Inside incrementByValue: " << x << endl;  // 输出：11
}

// 传引用参数
void incrementByReference(int& x) {
    x++;
    cout << "Inside incrementByReference: " << x << endl;  // 输出：11
}

int main() {
    int a = 10;

    // 传值调用
    incrementByValue(a);
    cout << "After incrementByValue: " << a << endl;  // 输出：10（a 未被修改）

    // 传引用调用
    incrementByReference(a);
    cout << "After incrementByReference: " << a << endl;  // 输出：11（a 被修改）

    return 0;
}

5. 总结

参数传递方式 特点 示例
传值参数 形参是实参的副本，修改形参不影响实参 void increment(int x) { ... }
传引用参数 形参是实参的别名，修改形参会影响实参 void increment(int& x) { ... }
  • 传值参数适用于不需要修改实参的场景。
  • 传引用参数适用于需要修改实参或避免复制的场景。
